# 냠킷 (Nyamkit) 서비스 기획서

## 1. 서비스 개요

- **서비스명**: 냠킷
- **서브카피**: 냠냠이의 먹킷리스트
- **컨셉**: 가성비 간식을 모아놓은 큐레이션 플랫폼
- **핵심**: "사라"가 아니라 "땡기는거 찾아"
- **타겟**: 밥 대신 간식으로 때우고 싶은 사람

---

## 2. 핵심 기능

### 2-1. 사용자 화면
| 기능 | 설명 |
|------|------|
| 메인 피드 | 주인장 픽 TOP, 랜덤 추천, 카테고리 미리보기 |
| 간식 카드 | 이름, 가격, 용량, 가성비 점수, 태그 표시 |
| 상세 페이지 | 준비 방법 + 구매 링크 (쿠팡 제휴) |
| 공유 버튼 | 카카오톡 / 링크 복사 공유 |
| 카테고리 페이지 | 냉동식품 / 과자 / 라면·즉석 / 음료 / 편의점 / 기타 |
| 랜덤 추천 | "오늘 뭐 먹지?" 버튼 |

### 2-2. 어드민 (데이터 입력)
| 기능 | 설명 |
|------|------|
| URL 자동 파싱 | 쿠팡 URL → 파트너스 API → 폼 자동완성 |
| 수동 입력 폼 | 자동완성 실패 시 직접 입력 |
| JSON 저장 | `/data/snacks.json`에 누적 저장 |
| 목록 관리 | 등록된 간식 확인 / 수정 / 삭제 |

## 3. 데이터 모델

```json
{
  "id": "snack_001",
  "name": "오뚜기 콤비네이션 피자",
  "short_desc": "에어프라이어로 바삭하게, 한 끼 해결",
  "category": "냉동식품",
  "price_approx": "약 4,000원대",
  "volume": "375g",
  "value_score": 4,
  "prep_type": "에어프라이어",
  "tags": ["에어프라이어", "주인장픽"],
  "prep": {
    "steps": ["180도 예열", "앞면 10분", "뒤집어서 5분"],
    "time_min": 15
  },
  "purchase_url": "https://link.coupang.com/...",
  "image_url": "https://...",
  "created_at": "2026-04-02"
}
```

> `prep` 필드는 선택사항 — 그냥 먹는 간식은 비워둠

### 카테고리 목록
`냉동식품` `과자` `라면·즉석` `음료` `편의점` `기타`

### 준비 방식 (prep_type)
`그냥먹기` `전자레인지` `에어프라이어` `끓이기` `전기밥솥`

### 태그 목록
`주인장픽` `신상` `혼밥` `야식` `든든함` `간단함`

### 가성비 점수 (value_score)
- 1~5점, 주인장이 직접 평가

### 가격 표기 방식
- 쿠팡 가격은 변동이 잦으므로 "약 4,000원대" 형식으로 기재
- 정확한 가격은 구매 링크에서 확인하도록 유도

## 4. URL 자동 파싱 흐름

```
1. 어드민에서 쿠팡 URL 입력
2. URL에서 상품 ID 추출 (/products/{productId})
3. 쿠팡 파트너스 API 호출
4. 상품명 / 가격 / 이미지 / 카테고리 자동완성
5. 조리법 / 가성비점수 / 태그 직접 입력
6. 저장 버튼 → snacks.json에 추가
```

### 환경변수
```
COUPANG_ACCESS_KEY=
COUPANG_SECRET_KEY=
```

---

## 5. UI/UX 방향

- **카드형 구조** - 이미지 + 가격 + 가성비 점수 한눈에
- **3초 UX** - 보자마자 땡기는지 판단 가능
- **정보 순서**: 이미지 → 가격 → 준비 방법 → 구매 링크
- **준비 방식 필터** - 그냥먹기 / 전자레인지 / 에어프라이어 등으로 필터링
- **공유 버튼** - 카카오톡 공유 / 링크 복사
- **모바일 우선** 레이아웃
